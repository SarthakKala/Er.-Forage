# Er. Forge — Demo Script

## Setup (do before the demo)
- Log in with the demo account (`demo@erforge.io`)
- Make sure sync has run recently (topbar badge)
- Have the shareable report URL ready to paste (seed script prints one)

## The Demo (5 minutes)

### 1. The problem (30 seconds)
“Every developer uses AI to fix bugs. But fixing a bug and learning from it are completely different things. Er. Forge is built for skill growth — not just solutions.”

### 2. Connect and sync (1 minute)
- Open onboarding
- Connect LeetCode (session + csrf)
- Start sync and show the progress

### 3. The analysis (1 minute)
- Open Submissions
- Click a **failed** submission
- Show the AI analysis:
  - root cause
  - missing mental model
  - concept tags (what skill was actually being tested)

### 4. The instructor (1 minute)
- Open Assignments
- “Er. Forge didn’t just say ‘learn hash maps’ — it assigned these exact two problems because that’s where the gap is.”
- Mention: completing an assigned problem is detected on the next sync

### 5. The portfolio (1 minute)
- Open Portfolio
- Show the growth timeline over weeks
- Generate a report
- Open the public link:
  - “This is what you show in your next interview.”

## Key talking points
- AI tools can reduce learning when they just give answers; Er. Forge is designed to build durable skill
- No other tool closes the full loop: observe → diagnose → prescribe → verify
- The shareable report is something GitHub and LeetCode can’t provide: structured improvement over time
